Ium Statistics System

  • Internal Project (unalbe to open system structure or source code to the public)
  • Made with Python (using Flask) and MongoDB
  • Before this system, there was statistics program written in JAVA using Play framework. However, this program was extremely slow and has high redundancy. It calculated whole statistics information every user access. I design new statistics system using python script and NoSQL (MongoDB) to reuse old calculated and stored data.
  • There are two reasons why I used NoSQL. First, for every 2 months, this system produce about 1GB statistics data. To reduce the cost of server, we use several cheap server machine to store it. RDB(Relational Database) system is not suitable for distributed storage system while NoSQL is. Second, statistics system should has affinity which means strict data schema which is for RDB is not suitable for this system.
  • New statistic program has two part: server-side and frontend-side. Server side part has script written by Python. Once in a day, scheduler (in this case I used Linux crontab) run scripts and scripts calculate the required statistics information and store them into mongoDB. Frontend-side part is web application using Flask. This application read the statistic data tuples from mongoDB and show it in chart format using hi-chart.
